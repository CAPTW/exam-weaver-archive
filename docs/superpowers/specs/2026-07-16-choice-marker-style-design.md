# 선지 번호 표시 방식 설계

## 목표

사용자가 앱 설정에서 객관식 선지 표기를 `㉮ ㉯ ㉴ ㉵`와 `① ② ③ ④` 중 선택하고, 선택한 표기가 문제 편집·문제 풀이·정답 표시·DOCX 본문과 정답표에 일관되게 나타나도록 한다.

## 결정한 접근법

표시 방식은 문제 데이터가 아니라 앱 전역 표시 설정으로 관리한다. DB의 `choice_number`를 의미상 기준값으로 사용하고, 기존 `choice_symbol`은 원본 호환을 위해 변경하지 않는다. 이 방식은 기존 문제은행과 Mounted 문제은행을 마이그레이션하지 않아도 되며, 설정을 바꿀 때 모든 문제에 즉시 같은 표기를 적용할 수 있다.

검토한 다른 접근법은 다음과 같다.

- 문제마다 선택 기호를 DB에 다시 저장: 기존 DB와 Mounted DB를 일괄 변경해야 하고 같은 문제의 표시가 앱 설정과 충돌하므로 제외한다.
- DOCX 출력 화면에만 별도 옵션 제공: 편집기와 문제 풀이의 표기가 달라져 전역 UX 요구를 충족하지 못하므로 제외한다.

## 구성 요소

1. `src/choice_markers.py`
   - 지원 스타일 식별자와 기본값을 정의한다.
   - 선지 번호를 선택한 스타일의 표시 문자로 바꾼다.
   - 잘못된 설정값은 기존 표기 방식으로 안전하게 정규화한다.
2. `src/gui/choice_marker_settings.py`
   - `data/app_settings.json`의 `choice_marker_style`을 읽고 쓴다.
   - 메뉴 언어 등 기존 설정 키를 보존한다.
3. 설정 창과 메인 창
   - 설정 창에 `선지 번호 표시` 콤보박스를 추가한다.
   - 메인 창은 저장된 값을 시작 시 로드하고 문제 관리·문제 풀이·모의고사 출력 화면에 주입한다.
   - 설정 적용 시 이미 열린 문제 편집창과 진행 중인 문제 풀이 화면도 다시 표시한다.
4. 문제 편집기
   - 행 라벨, 입력 안내, 정답 콤보박스와 관련 도움말에 선택 스타일을 사용한다.
   - 저장할 때는 기존 DB의 `choice_symbol`을 보존하고 새 선지만 기존 호환 기호로 저장한다.
5. 문제 풀이
   - 선지 버튼과 채점 피드백의 정답/선택 표시에 선택 스타일을 사용한다.
6. DOCX 출력
   - 저장된 원본 기호 대신 `choice_number`와 전역 스타일로 본문 선지와 정답표를 렌더링한다.
   - 선지 순서 섞기 후에도 새 번호에 맞는 표시와 정답표를 생성한다.

## 기본값과 호환성

- 기본값은 기존 동작인 `㉮ ㉯ ㉴ ㉵`이다.
- 숫자 원문자는 편집기가 지원하는 최대 10개 선지까지 `①`~`⑩`으로 표시한다.
- 알 수 없는 스타일 값, 범위를 벗어난 번호, 번호가 없는 비정상 데이터는 기존 저장 기호 또는 숫자 문자열로 안전하게 폴백한다.
- DB 스키마와 문제은행 파일은 변경하지 않는다.

## 오류 처리

- 설정 파일이 없거나 JSON이 손상되면 기존 표기 방식으로 시작한다.
- 설정 저장 실패 시 기존 앱 설정과 현재 UI 값을 유지하고 오류 알림을 보여준다.
- DOCX 출력은 유효한 번호가 있으면 선택 스타일을 우선하고, 그렇지 않으면 원본 기호를 사용한다.

## 테스트

- 표시 변환: 두 스타일, 1~10, 잘못된 스타일과 범위 밖 번호.
- 설정 저장: 기존 메뉴 언어 보존, 재실행 로드, 손상/미지원 값 폴백.
- 설정 UI: 두 항목 노출과 선택값 반환.
- 편집기: 숫자 원문자 표시 및 DB 원본 기호 보존.
- 문제 풀이: 선지와 정답 피드백에 숫자 원문자 사용.
- DOCX: 본문과 정답표에 `① ② ③ ④`가 들어가며 `㉮ ㉯ ㉴ ㉵`가 표시되지 않는지 확인.
- 관련 GUI·DOCX·설정 회귀 테스트와 전체 테스트를 실행한다.

# 모의고사 출력 과목 적용 범위 UX 설계

## 배경

현재 메뉴와 DOCX 출력 화면은 `시험지 출력`으로 표시되지만, 실제 작업은
기출문제를 선택해 모의고사 DOCX를 만드는 흐름이다. 또한 `전체 과목에 적용`은
모든 과목을 사용 선택하고 같은 문항 수를 설정하므로, 이미 선택한 일부 과목의
문항 수만 일괄 변경하려면 행별로 수정해야 한다.

## 목표

- 한국어 메뉴와 화면 제목을 `모의고사 출력`으로 통일한다.
- `전체 과목에 적용` 옆에 `선택한 과목에만 적용` 버튼을 추가한다.
- 새 버튼은 표의 `사용` 체크박스가 체크된 행의 문항 수만 변경한다.
- 기존 단일 시험, 여러 시험 조합, 중복 제거, 연도·해시태그 필터와 DOCX 생성
  로직은 변경하지 않는다.

## 화면과 용어

- 한국어 메뉴: `시험지 출력` → `모의고사 출력`
- 화면 제목: `시험지 출력 (DOCX)` → `모의고사 출력 (DOCX)`
- 영어 메뉴 팩: `Export Exam` → `Mock Exam Export`
- 문항 수 입력, `전체 과목에 적용`, `선택한 과목에만 적용`을 한 행에
  배치하고 두 적용 범위를 즉시 비교할 수 있게 한다.

## 상호작용

### 전체 과목에 적용

기존 동작을 유지한다. 클릭하면 현재 표의 모든 과목을 `사용` 상태로 바꾸고,
`과목당 문항 수`의 값을 모든 행에 설정한다.

### 선택한 과목에만 적용

- `사용` 체크박스가 이미 체크된 행에만 `과목당 문항 수`를 설정한다.
- 미선택 행의 체크 상태와 기존 문항 수는 변경하지 않는다.
- 선택한 과목이 하나도 없으면 버튼을 비활성화해 의미 없는 클릭을 방지한다.
- 과목 선택이 바뀔 때마다 버튼 활성 상태와 `선택 N과목 · 예상 N문항` 요약을
  함께 갱신한다.
- 단일 시험 모드와 여러 시험 조합 모드에서 동일하게 동작한다.

## 구현 경계

`ExportInterface` 내에 선택 행의 개수를 반환하는 적용 메서드를 둔다. 행 데이터의
`checkbox`와 `count_spin` 참조를 재사용하고, Repository나 DOCX 내보내기 계층에는
새로운 상태나 분기를 추가하지 않는다. 메뉴 route 키 `menu.export`도 유지해 저장된
언어 설정과 외부 언어 팩의 호환성을 보존한다.

## 오류 및 경계 처리

- 행이 없는 상태와 선택한 행이 없는 상태는 새 버튼 비활성화로 처리한다.
- 버튼 signal이 `False`를 인자로 넘기는 PyQt 동작을 정수 문항 수로 오인하지 않고
  `allSubjectCountSpin`의 값을 사용한다.
- `count_spin`이 없는 비정상 행은 건너뛰어 다른 행에 영향을 주지 않는다.

## 테스트

- 한국어 메뉴 팩과 화면 제목이 `모의고사 출력`으로 표시된다.
- 한국어 fallback과 기본 한국어 언어 팩이 새 명칭을 사용한다.
- 영어 메뉴 팩이 `Mock Exam Export`를 사용한다.
- 전체 적용은 모든 행을 선택하고 같은 문항 수로 바꾸는 기존 회귀를 유지한다.
- 선택 적용은 체크된 행의 문항 수만 바꾸고 미선택 행을 그대로 둔다.
- 버튼 signal 인자가 있을 때도 스핀 박스의 값을 사용한다.
- 선택이 없으면 선택 적용 버튼이 비활성화되고, 하나 이상 선택하면 활성화된다.
- 기존 단일·복수 시험 DOCX 출력 및 중복 제거 회귀 테스트가 계속 통과한다.

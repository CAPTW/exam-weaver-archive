# 문제 수정창 이미지 작업 및 다중 창 설계

## 목표

문제 관리 화면의 개별 문제 수정창에서 이미지 작업 버튼을 겹치지 않게 분리하고, 현재 문제 이미지를 시스템 클립보드로 복사할 수 있게 한다. 서로 다른 문제는 각각 별도 수정창에서 동시에 편집할 수 있게 하되, 같은 문제는 하나의 수정창만 유지한다.

## 범위

- 문제 이미지 영역에 `이미지 변경`, `붙여넣기`, `클립보드 복사`, `삭제` 버튼을 각각 독립된 버튼으로 제공한다.
- 이미지 상태 표시와 버튼 행에 충분한 레이아웃 높이를 보장해 버튼이 겹치거나 잘리지 않게 한다.
- 현재 이미지 파일을 `QImage`로 읽어 `QApplication.clipboard()`에 복사한다.
- 문제 관리 화면에서 연 개별 문제 수정창만 비모달로 운용한다.
- 문제 추가, 서술형 추가, 복제 대화상자는 기존 모달 동작을 유지한다.
- 서로 다른 문제 수정창을 동시에 열 수 있다.
- 같은 문제의 수정 버튼을 다시 누르면 기존 창을 앞으로 가져온다.

선지 이미지 제어는 이번 변경 범위에 포함하지 않는다.

## 이미지 UI 및 상태

문제 이미지 미리보기 옆 제어 영역은 상태 표시와 단일 수평 버튼 행으로 구성한다. 버튼은 고정 간격을 두며 이미지 영역의 높이를 버튼 행과 상태 표시가 모두 들어가도록 확보한다.

`클립보드 복사` 버튼은 현재 이미지 경로가 존재하고 읽을 수 있을 때만 활성화한다. 이미지 변경이나 붙여넣기 성공 시 활성화하고, 삭제하거나 파일을 읽을 수 없으면 비활성화한다. 복사 실행 중 이미지가 사라졌거나 읽기에 실패하면 상태 문구로 실패를 알리고 버튼을 비활성화한다. 복사는 원본 파일을 변경하거나 새 파일을 만들지 않는다.

## 다중 수정창 수명 관리

`BrowserInterface`는 열린 수정창을 문제 ID별 레지스트리에 보관한다. `open_editor(question_id)`는 다음 순서로 동작한다.

1. 같은 문제 ID의 살아 있는 창이 있으면 그 창을 복원하고 앞으로 가져온 뒤 포커스를 준다.
2. 없으면 현재 Repository에서 문제와 과목 정보를 읽어 `QuestionEditor`를 생성한다.
3. 이 수정창만 비모달로 설정하고 `show()`한다.
4. 창이 승인되면 창을 열 당시의 Repository에 수정 데이터를 저장한다.
5. 저장 성공 시 현재 목록 또는 검증 결과를 갱신하고 성공 알림을 표시한다.
6. 저장 실패 시 오류 알림을 표시한다.
7. 창이 닫히면 레지스트리에서 해당 항목을 제거한다.

창을 열 때 Repository 참조를 저장 콜백에 함께 캡처한다. 사용자가 편집 중 Mount 구성을 새로 고쳐도 해당 편집창은 처음 읽었던 DB를 저장 대상으로 유지한다.

## 데이터와 충돌 처리

동일 문제의 중복 창을 허용하지 않으므로 같은 앱 인스턴스 안에서 발생하는 마지막 저장 우선 충돌을 방지한다. 서로 다른 문제는 독립적으로 저장된다. 외부 프로그램이나 다른 앱 인스턴스에서 같은 레코드를 바꾼 경우를 감지하는 버전 잠금은 이번 범위에 포함하지 않는다.

## 테스트

- 네 개의 문제 이미지 버튼이 모두 존재하고 독립된 수평 레이아웃에 배치되는지 검증한다.
- 유효한 현재 이미지를 클립보드에 복사했을 때 이미지 크기와 픽셀 데이터가 유지되는지 검증한다.
- 이미지 없음, 삭제, 읽기 실패 시 복사 버튼이 비활성화되는지 검증한다.
- 서로 다른 두 문제를 열면 두 비모달 수정창이 동시에 관리되는지 검증한다.
- 같은 문제를 다시 열면 새 창을 만들지 않고 기존 창을 활성화하는지 검증한다.
- 각 창의 승인 신호가 올바른 문제 ID와 처음 캡처한 Repository를 대상으로 저장되는지 검증한다.
- 저장 성공 후 문제 목록 또는 검증 결과가 갱신되고 창 종료 후 레지스트리가 정리되는지 검증한다.
- 관련 GUI 테스트와 전체 테스트 스위트를 실행해 회귀가 없는지 확인한다.

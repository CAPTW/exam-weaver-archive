# `<보기>` 한 셀 표 변환 및 편집 설계

## 목표

발문에 실제 보기 블록이 포함된 문항은 보기 블록을 `1행 × 1열` 표로 보존한다. 사용자는 문제 수정 화면에서 이 표를 확인·편집·추가·삭제할 수 있고, 신규 파싱과 기존 문제은행 이관 및 DOCX 출력에서도 같은 구조가 유지되어야 한다.

## 확인된 현재 상태

- 현재 `data/exam_bank.db`에는 `<보기>` 계열 표식이 있는 문항이 75개 있다.
- 75개 모두 `question_format_json.tables`가 없는 일반 텍스트 상태다.
- 65개는 발문 안에 참조용 표식과 실제 블록 표식이 각각 있어 표식이 두 번 나타난다.
- 10개는 실제 블록 표식만 한 번 나타난다.
- 현재 편집기는 이미 저장된 `tables`만 카드로 만들며, 표를 새로 추가하거나 일반 텍스트의 `<보기>` 블록을 표로 승격하는 기능이 없다.

## 선택한 접근법

공통 변환기를 하나 두고 파서, 편집기, DB 이관 도구가 동일한 규칙을 사용한다.

1. 발문에서 마지막 `<보기>` 계열 표식을 실제 보기 블록 시작점으로 선택한다.
2. 마지막 표식부터 발문 끝까지를 한 셀의 내용으로 옮긴다.
3. 표식 앞의 발문은 일반 텍스트로 유지한다.
4. 보기 블록 평문은 발문에서 제거하고 원래 위치를 표 anchor로 저장해 DOCX 중복 출력을 막는다.
5. 이미 동일한 보기 표가 저장된 문항은 다시 변환하지 않는다.

검토에서 제외한 방식은 다음과 같다.

- 수동 `표 추가`만 제공: 기존 75개와 신규 파싱 문항이 계속 일반 텍스트로 남는다.
- 편집창을 열 때만 자동 변환: 열지 않은 문제은행 문항과 직접 DOCX로 출력하는 문항은 개선되지 않는다.

## 표식과 변환 경계

인식 대상은 공백 변형을 허용한 `<보기>`, `＜보기＞`, `〈보기〉`, `《보기》`, `[보기]`, `【보기】`다. 여러 표식이 있으면 마지막 표식을 사용하므로 `다음 <보기>에서 ... <보기> ① ...`의 첫 번째 참조용 표식은 일반 발문에 남는다.

한 셀에는 마지막 표식과 뒤쪽 내용 전체가 들어간다. 원문 표식은 손실 없이 보존하고 표식과 내용 사이에는 표시용 줄바꿈을 둔다. 표식 뒤에 텍스트가 없고 보기 내용이 문제 이미지로만 존재하는 문항도 표식 자체를 한 셀 표로 변환하며 기존 이미지는 유지한다.

## 데이터 모델

기존 schema-v2 `question_format_json.tables`를 그대로 사용한다. 합성된 보기 표는 원본 이미지가 없는 편집 가능한 표이므로 `render_mode`와 `recommended_render`를 `native`로 둔다.

```json
{
  "schema_version": 2,
  "tables": [
    {
      "id": "view-table-1",
      "rows": [["<보기>\n① 첫 번째 설명 ② 두 번째 설명"]],
      "cells": [
        {
          "row": 0,
          "col": 0,
          "text": "<보기>\n① 첫 번째 설명 ② 두 번째 설명",
          "row_span": 1,
          "col_span": 1,
          "horizontal_alignment": "left",
          "vertical_alignment": "center"
        }
      ],
      "anchor": {
        "offset": 18,
        "before_context": "옳은 것을 모두 고른 것은?",
        "after_context": ""
      },
      "source": {
        "kind": "view_block_text"
      },
      "confidence": {
        "score": 1.0,
        "reasons": ["explicit_view_marker"]
      },
      "render_mode": "native",
      "recommended_render": "native"
    }
  ]
}
```

기존 밑줄 span, 다른 표, 알 수 없는 확장 필드는 그대로 보존한다. ID는 기존 ID와 충돌하지 않는 `view-table-N`을 선택한다.

## 문제 수정 화면

발문 도구 모음에 `발문 표 추가`를 배치하고 표 영역은 표가 없어도 추가 버튼이 보이도록 유지한다.

- 텍스트를 선택하고 추가: 선택한 텍스트를 발문에서 제거하고 그 위치에 한 셀 표로 삽입한다.
- 선택 없이 추가: 현재 커서 위치에 `<보기>`가 들어 있는 빈 한 셀 표를 삽입한다.
- 구조 보기: 한 셀 내용을 읽기 전용으로 확인한다.
- 표 편집: 기존 `QTableWidget` 편집기를 사용해 셀 내용을 변경한다.
- 표 삭제: 표 내용을 anchor 위치의 발문으로 복원한 뒤 payload에서 표를 제거한다.
- 출력 방식: 합성 표 기본값은 `편집 가능한 표`지만 기존 자동·이미지·native 선택 UI는 유지한다.

기존 일반 텍스트 `<보기>` 문항을 편집기로 열면 저장 전 메모리에서 공통 변환기를 적용해 바로 표 카드가 보이게 한다. 사용자가 취소하면 DB는 바뀌지 않고, 저장하면 구조화된 표가 저장된다.

## 파서 연동

문항 텍스트 정리, OCR 문자 복구, LaTeX 정규화와 기하학 표 평문 제거가 끝난 뒤 공통 변환기를 적용한다. 이미 기하학 표가 `<보기>` 내용을 소유하여 평문을 제거한 경우에는 표식이 남지 않으므로 중복 표가 생성되지 않는다.

파서 결과의 `Question.text`에는 보기 블록이 빠진 일반 발문만 남고 `Question.format_json`에는 한 셀 표와 anchor가 들어간다. 이후 repository 저장 경로는 기존 형식 컬럼을 그대로 사용한다.

## 기존 문제은행 staging 이관

전용 이관 도구는 원본 SQLite를 직접 수정하지 않는다.

1. SQLite backup API로 별도 staging DB를 만든다.
2. staging 안에서 대상 문항을 한 트랜잭션으로 변환한다.
3. `PRAGMA integrity_check`, `PRAGMA foreign_key_check`, 문항/선지 수와 비대상 열 해시를 검증한다.
4. 변환 전후 발문을 조합했을 때 원래 `<보기>` 블록이 손실되지 않았는지 검사한다.
5. 재실행 결과 변경 건수가 0인지 확인한다.
6. 검증된 staging만 원본 백업 후 원자적으로 현재 문제은행과 교체한다.

Mounted 문제은행에도 같은 도구를 적용할 수 있지만 경로를 명시적으로 전달해야 하며, 읽기 전용 표시가 있는 문제은행은 교체하지 않는다.

## DOCX 출력

기존 anchor 기반 표 출력기를 그대로 사용한다. 발문 앞부분을 먼저 출력하고 anchor에서 한 셀 Word 표를 추가한다. 보기 블록은 `question_text`에서 제거되어 있으므로 DOCX에는 한 번만 나타난다.

합성 보기 표는 이미지 소스가 없더라도 고신뢰 구조 표이므로 자동 모드에서 native Word 표로 출력한다. 사용자가 문서 전체를 이미지 모드로 지정해도 원본 이미지가 없으면 기존 fallback에 의해 native 표가 사용된다.

## 검증 전략

- 단일 표식, 이중 표식, 괄호 변형과 공백 변형
- 이미지 보기 문항의 내용 없는 마지막 표식도 한 셀 표로 변환
- 기존 span과 다른 표 보존
- 동일 입력 재실행의 무변경
- 편집창 자동 승격, 선택 텍스트 표 추가, 빈 표 추가, 편집, 삭제/원문 복원
- 파서 결과에서 발문/표 분리
- DOCX OpenXML에서 `w:tbl` 존재와 `<보기>` 중복 없음
- staging DB에서 75개 대상 변환, 모든 ID·정답·선지·이미지·문항 수 유지
- staging 변환 재실행 시 0개 변경
- 전체 pytest 회귀

## 완료 조건

- 기존 문제은행의 75개 `<보기>` 문항이 검증된 한 셀 표로 이관된다.
- 신규 파싱 문항도 동일 규칙으로 자동 저장된다.
- 문제 수정 화면에서 표가 없어도 추가할 수 있고 기존 표를 확인·편집·삭제할 수 있다.
- 표 삭제 시 내용이 발문에 복원되어 데이터가 사라지지 않는다.
- DOCX에서 보기 블록이 한 셀 표로 정확히 한 번 출력된다.
- 비대상 문항과 정답·선지·이미지 데이터에 회귀가 없다.

# 한국어 중심 UI 및 메뉴 언어 팩 설계

## 배경

현재 앱은 화면별 한글화 수준이 다르다. 문제 풀이와 문제 가져오기는 대부분
한글이지만, 시험지 출력은 제목·필터·표 머리글·알림이 영문 중심이고 문제은행
연결 관리에는 `Mount`, `Source`, `Target`, `Dry-run` 같은 구현 용어가 노출된다.
문제 관리의 `EXAM`·`SUBJECT`, Codex 패널의 `Model`·`You`·`System`·`Error`도
화면 전체의 언어 일관성을 해친다. 여러 시험의 과목을 한 DOCX로 내보내는
기능처럼 이미 구현된 동작도 영문 체크박스와 낮은 시각적 우선순위 때문에
발견하기 어렵다.

이번 변경은 사용자 노출 UI를 한국어 중심으로 통일하고, 기능의 의미와 다음
행동이 즉시 이해되도록 배치·설명·상태 안내를 미세 조정한다. 전체 화면을
다국어화하지는 않지만, 메인 메뉴는 한국어와 영어를 전환할 수 있는 독립적인
언어 팩 구조를 제공한다.

## 목표

- 메인 메뉴, 전체 업무 화면, 편집창, 대화상자, 알림의 용어와 문체를
  자연스러운 한국어로 통일한다.
- 사용자 화면의 `DB`는 `문제은행`, `태그`는 `해시태그`로 표시한다.
- 버튼은 대상보다 행동이 먼저 이해되는 표현으로 통일한다.
- 화면마다 제목, 설명, 조건 입력, 결과, 주요 실행 동작의 위계를 일관되게
  만든다.
- 단일 시험과 여러 시험 조합 등 중요한 기능의 현재 모드와 결과 범위를
  화면에서 바로 확인할 수 있게 한다.
- 오류 메시지는 발생 사실, 원인, 사용자가 취할 조치를 함께 제공한다.
- 메인 메뉴에 한정하여 한국어·영어 기본 팩과 JSON 기반 추가 언어 팩을
  지원한다.
- Repository 주입, 연결된 문제은행 갱신, 쓰기 대상 선택, DOCX 생성 등 기존
  업무 로직을 보존한다.

## 비목표

- 화면 본문, 버튼, 대화상자, 알림 전체의 다국어 전환.
- 문제은행 스키마, 문항 데이터, 연결 manifest 또는 파일 형식 변경.
- 앱의 전면적인 시각 디자인 교체.
- 기존 Repository 및 내보내기 엔진의 재구현.
- 사용자 언어 팩에 실행 코드나 스타일 재정의 기능 제공.

## 공통 용어

사용자에게 노출되는 문구는 다음 기준을 따른다.

| 기존 표현 | 개선 표현 |
| --- | --- |
| DB Mount | 문제은행 연결 관리 |
| DB / Database | 문제은행 |
| Source | 원본 문제은행 |
| Target | 대상 문제은행 |
| Exam | 시험 종류 |
| Subject | 과목 |
| Year range | 출제 연도 범위 |
| Tag / Hashtag / 태그 | 해시태그 |
| Dry-run | 연결 사전 검사 |
| Export | DOCX로 내보내기 |
| Model | 모델 |
| You / System / Error | 사용자 / 시스템 / 오류 |

`Codex`, `PDF`, `DOCX`, `OCR`, `ID`, 모델명과 파일 확장자는 고유명사 또는
표준 기술 표기로 유지한다. 클래스명, 변수명, 테이블명, manifest 필드처럼
사용자에게 직접 노출되지 않는 내부 식별자는 변경하지 않는다.

## 공통 화면 및 상호작용 원칙

각 업무 화면은 상단의 한글 제목과 한 줄 설명, 중단의 검색·필터·편집 영역,
결과 영역, 주요 실행 동작 순으로 구성한다. 화면당 주 실행 버튼은 하나만
강조하고 보조 동작은 중립 스타일을 사용한다. 삭제·덮어쓰기·문제은행 교체처럼
되돌리기 어려운 작업은 별도 위험 스타일과 확인 대화상자를 사용한다.

버튼은 `조회`, `문제 추가`, `오류 검사`, `DOCX로 내보내기`, `저장`, `취소`처럼
동작을 직접 표현한다. 모호한 `확인`은 실제 결과에 따라 `저장`, `적용`, `삭제`
등으로 바꾼다. 비활성화된 컨트롤에는 이유를 툴팁으로 제공하며, 색상만으로
상태를 구분하지 않는다.

Tab 이동 순서는 화면의 시각적 순서와 일치시킨다. 버튼 크기와 간격을
통일하고 Windows 화면 배율 125%에서도 문구가 잘리지 않도록 최소 너비와
레이아웃 확장 우선순위를 지정한다. 결과가 없을 때는 빈 표만 보여주지 않고
다음에 취할 행동을 안내한다.

## 화면별 설계

### 메인 메뉴

기본 한국어 메뉴는 `홈`, `문제 관리`, `문제 풀이`, `시험지 출력`,
`문제 가져오기`, `문제은행 연결 관리`, `Codex`, `설정` 순서로 표시한다.
`Codex`는 브랜드명으로 유지한다. 메뉴 텍스트만 선택된 메뉴 언어 팩의 영향을
받고 각 화면의 본문은 한국어를 유지한다.

### 문제 관리

- 첫째 행에는 `시험 종류`, `과목` 필터와 문제 추가·검사·삭제 동작을 둔다.
- 둘째 행에는 넓은 `해시태그 또는 문제 내용 검색` 입력창과 바로 옆 `조회`
  버튼을 둔다.
- 조회 결과 수와 현재 연결된 문제은행을 표시한다.
- 버튼은 `문제 추가`, `서술형 문제 추가`, `오류 검사`, `선택 문제 삭제`로
  표시한다.
- 행별 `수정`, `복제`, `삭제`, `해설` 버튼의 위치와 크기를 통일한다.

### 문제 풀이

- 조건은 `시험 종류`, `출제 연도 범위`, `해시태그 필터`, `과목별 출제`로
  표시한다.
- 현재 문제은행과 시험 종류를 상단에서 확인할 수 있게 한다.
- 선택 조건으로 출제 가능한 문항 수를 시험 시작 전에 안내한다.
- 결과가 없으면 출제 연도, 해시태그 또는 과목 조건 중 무엇을 조정할 수
  있는지 안내한다.

### 시험지 출력

- 기존의 발견하기 어려운 영문 체크박스 대신 `구성 방식`을 독립된 선택
  영역으로 노출한다.
- 구성 방식은 `한 시험에서 구성`과 `여러 시험의 과목을 조합` 두 가지다.
- 여러 시험 조합에서는 표에 `문제은행`, `시험 종류`, `과목`, `문항 수`를
  표시한다.
- 출제 연도 범위와 해시태그가 모든 선택 과목에 공통 적용됨을 짧게 설명한다.
- 하단에 선택 과목 수와 예상 총 문항 수를 요약한다.
- 주 실행 버튼은 `DOCX로 내보내기`로 표시한다.
- 기존 단일 시험과 복수 시험 선택·검증·중복 제거·DOCX 생성 경로를 그대로
  재사용한다.

### 문제 가져오기

- `파일 선택 → 분석 및 검수 → 문제은행에 저장` 단계를 화면에서 확인할 수
  있게 한다.
- OCR, 파싱, 품질 검사 결과를 구분해 표시한다.
- 저장 전 정상 문항, 자동 교정 문항, 검토 필요 문항 수를 요약한다.

### 문제은행 연결 관리

- 화면 제목과 메뉴를 `문제은행 연결 관리`로 표시한다.
- 필드는 `원본 문제은행`, `대상 문제은행`, `시험 종류`로 통일한다.
- 동작은 `연결 사전 검사`, `연결 설정 저장`, `문제은행 내보내기`,
  `시험 사본 만들기`로 표시한다.
- 각 연결의 활성 상태와 쓰기 가능 여부를 텍스트가 포함된 상태 배지로
  보여준다.
- 실패 시 문제은행 파일, 실패 원인, 권장 조치를 안내한다.

### 문제 편집창

- 작업에 따라 제목을 `문제 추가`, `서술형 문제 추가`, `문제 수정`으로
  표시한다.
- 입력 순서는 `기본 정보 → 발문 → 선지 → 정답 → 해설 → 해시태그`다.
- 이미지 동작은 `이미지 변경`, `붙여넣기`, `클립보드에 복사`, `삭제`로
  분리한다.
- 동일한 문제의 편집 요청은 기존 창을 앞으로 가져오고 서로 다른 문제는
  별도 창에서 동시에 편집하는 현재 동작을 유지한다.
- 저장하지 않고 닫을 때 변경 내용 폐기를 확인한다.
- 저장 실패 시 입력 내용을 유지하고 원인을 표시한다.

### Codex 패널

- `Codex`와 모델명은 유지한다.
- `Model`, `You`, `System`, `Error`는 `모델`, `사용자`, `시스템`, `오류`로
  표시한다.
- 인증, 로그인, 새 작업, 보내기 버튼의 문체와 크기를 앱 전체와 통일한다.
- 연결, 대기, 실행, 완료, 실패 상태를 색상과 한글 문구로 함께 표시한다.
- 긴 기술 오류는 기본 메시지 아래 접을 수 있는 `상세 정보`에 배치한다.

## 메뉴 언어 팩

### 제공 범위

한국어(`ko`)와 영어(`en`) 메뉴 팩을 기본 제공한다. 사용자는 설정에서
`메뉴 언어`를 선택할 수 있으며 선택 결과는 즉시 메인 메뉴에 반영되고 다음
실행에도 유지된다. 화면 본문, 버튼, 알림, 로그는 선택과 무관하게 한국어를
사용한다.

### 팩 형식

메뉴 언어 팩은 표시 문자열만 가진 UTF-8 JSON 파일이다. 팩은 언어 코드,
표시 이름, 버전, 문자열 맵을 포함한다. 문자열 키는 안정적인 내부 route와
분리한다.

```json
{
  "locale": "en",
  "display_name": "English",
  "version": 1,
  "strings": {
    "menu.home": "Home",
    "menu.question_management": "Question Management",
    "menu.practice": "Practice",
    "menu.export": "Export Exam",
    "menu.import": "Import Questions",
    "menu.question_bank_connections": "Question Bank Connections",
    "menu.codex": "Codex",
    "menu.settings": "Settings"
  }
}
```

기본 팩은 앱 리소스에 포함한다. 사용자가 추가하는 팩은 쓰기 가능한
`data/language_packs/menu` 폴더에서 발견한다. 외부 팩은 알려진 메뉴 키의
문자열 값만 읽으며 코드, 경로, 스타일 또는 임의 객체를 실행하거나 적용하지
않는다.

선택한 팩에 키가 없거나 값이 유효하지 않으면 해당 항목만 한국어 기본값으로
대체한다. 팩 전체가 손상된 경우 해당 팩을 선택 목록에서 제외하고 설정 화면에
간단한 경고를 표시한다. 저장된 locale을 찾을 수 없으면 한국어로 시작한다.

### 설정 화면

현재 동작이 연결되지 않은 하단 `설정` 메뉴는 작은 설정 대화상자를 연다.
초기 설정 항목은 `메뉴 언어` 하나이며 발견된 유효 언어 팩을 콤보 상자에
표시한다. 사용자가 `적용`을 누르면 선택을 휴대 가능한 앱 설정 파일에
원자적으로 저장하고 메인 메뉴 텍스트를 다시 적용한다. `취소`는 저장된
언어와 현재 메뉴를 변경하지 않는다.

설정 대화상자 자체와 오류 메시지는 메뉴 언어의 적용 범위가 아니므로
한국어로 유지한다.

## 데이터 흐름

앱 시작 시 UI 설정을 읽고 기본·외부 메뉴 팩을 발견·검증한 뒤 저장된 locale을
선택한다. MainWindow는 메뉴 route별 위젯 참조와 번역 키를 보관하고 선택된
문자열을 적용한다. 설정 변경 시 검증된 locale만 저장하고 동일한 적용 함수를
호출한다. 메뉴 언어 변경은 stacked interface, Repository, 문제은행 연결,
필터 상태 또는 편집창에 영향을 주지 않는다.

각 업무 화면의 한국어 문구 개선은 기존 signal과 slot, Repository 호출,
검증기, 내보내기 엔진에 연결된 컨트롤을 재사용한다. 시험지 출력의 구성 방식
표현은 기존 단일/복수 시험 boolean 상태에 연결하고 별도 선택 엔진을 만들지
않는다.

## 알림 및 오류 처리

알림 제목은 `완료`, `확인 필요`, `입력 오류`, `처리 실패`를 기본으로 한다.
본문은 발생한 일, 원인, 사용자가 취할 조치 순으로 작성한다. 파일 작업 실패
시 대상 경로와 재시도 방법을 표시한다. 성공 알림은 짧게 표시하고 자동으로
닫히며 작업 흐름을 막지 않는다.

삭제, 덮어쓰기, 문제은행 교체에는 확인 대화상자를 적용한다. 언어 팩 오류는
앱 시작을 막지 않으며 한국어 fallback으로 복구한다. UI 문구 변경 때문에
Repository 또는 파일 작업 예외가 삼켜지지 않도록 기존 예외 경로를 유지한다.

## 테스트 전략

자동 테스트는 다음을 검증한다.

- 메인 메뉴와 모든 업무 화면에서 합의하지 않은 사용자 노출 영문 문구가
  남지 않는다.
- 사용자 화면의 `DB`와 `태그`가 각각 `문제은행`과 `해시태그`로 바뀐다.
- 문제 관리 검색 입력창이 독립된 둘째 행을 채우고 `조회` 버튼이 바로 옆에
  배치된다.
- 문제 풀이와 시험지 출력에서 연결된 문제은행의 모든 시험 종류가 유지된다.
- 시험지 출력의 두 구성 방식이 명시적으로 보이고 기존 단일/복수 시험 상태와
  동일하게 동작한다.
- 서로 다른 시험의 과목을 선택해 실제 DOCX를 생성했을 때 과목별 문항 수,
  공통 연도 범위와 중복 제거가 유지된다.
- 문제 편집창의 이미지 변경·붙여넣기·복사·삭제와 다중 창 동작이 유지된다.
- 문제은행 연결·교체와 연결된 문제 수정·삭제 등 쓰기 작업이 유지된다.
- 한국어와 영어 팩을 선택하면 메뉴 항목만 즉시 변경되고 설정이 재시작 후에도
  유지된다.
- 누락 키, 잘못된 타입, 손상된 JSON, 알 수 없는 저장 locale은 한국어
  fallback으로 안전하게 복구된다.
- 외부 언어 팩은 알려진 문자열 키 외의 값을 적용하지 않는다.
- `Run_Latest_App.bat` 실행 결과가 소스 최신 상태와 일치한다.
- 일반 배율과 Windows 125% 배율에서 주요 컨트롤과 문구가 잘리지 않는다.
- 전체 자동 회귀 테스트가 통과한다.

## 완료 조건

기본 한국어 상태에서 메인 메뉴부터 모든 주요 업무 화면, 편집창, Codex 패널,
알림까지 합의한 용어와 행동 중심 문구를 사용한다. 여러 시험 조합과
문제은행 연결 같은 기존 기능은 사용자가 별도 설명 없이 찾을 수 있고 기존
데이터 동작은 변하지 않는다. 설정에서 English를 선택하면 메인 메뉴 항목만
영어로 즉시 전환되고 앱 재시작 뒤에도 유지되며, 한국어로 되돌릴 수 있다.
추가 JSON 메뉴 팩은 앱 코드 변경 없이 발견되며 누락 또는 손상 시 한국어로
안전하게 대체된다.

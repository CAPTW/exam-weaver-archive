# 다중 셀 표 편집기 및 DOCX 자동 배치 설계

## 목표

문제 수정 화면에서 사용자가 표의 행과 열을 독립적으로 추가·삭제하고, 셀을 병합·분할하며, 열 너비를 직접 조정할 수 있게 한다. 수동 너비가 없는 표는 DOCX 출력 시 셀 내용과 페이지 가용 폭을 분석해 열 너비와 2단·1단 배치를 자동 결정한다.

기존 `1행 × 1열` `<보기>` 표, PDF에서 파싱한 다중 셀 표, schema-v2 형식 payload와 완전히 호환되어야 한다.

## 현재 구조의 한계

현재 문제 편집기는 저장된 `rows` 크기로 `QTableWidget`을 만들고 셀 텍스트만 수정한다. 새 표는 항상 `1행 × 1열`이며 다음 동작을 지원하지 않는다.

- 행·열 추가 및 삭제
- 셀 병합 및 분할
- 열 너비 드래그와 저장
- 구조 변경 중 기존 병합 영역 보호

DOCX exporter는 저장된 `column_widths`가 있을 때만 총 82mm 기준으로 폭을 적용한다. 폭 데이터가 없으면 내용 기반 계산을 하지 않고 Word 기본 동작에 맡기며, 5열 이상이라는 단순 조건으로만 1단 전환을 결정한다.

## 선택한 접근법

기존 `QTableWidget` 편집기를 확장한다.

1. 새 표는 계속 `1행 × 1열`로 만든다.
2. 선택 셀 기준으로 행과 열을 독립적으로 추가한다.
3. 선택한 행·열을 삭제하고 최소 `1행 × 1열`을 보장한다.
4. 직사각형 선택 영역을 병합하고 병합 시작 셀에서 분할한다.
5. 열 헤더 드래그 결과를 정규화된 비율로 저장한다.
6. DOCX exporter는 수동 폭, 파싱 원본 폭, 내용 기반 자동 폭 순으로 레이아웃을 선택한다.

검토에서 제외한 방식은 다음과 같다.

- 스프레드시트형 전용 모델·delegate를 새로 구현: 장기 확장성은 높지만 현재 요구보다 범위가 크고 기존 편집기와 중복된다.
- HTML 표 편집기: 자유도는 높지만 PyQt와 schema-v2 및 Word 표 사이의 변환 오차가 커진다.

## 편집 UX

### 새 표

`발문 표 추가`는 현재처럼 선택 텍스트를 표로 옮기거나 커서 위치에 기본 `<보기>` 셀을 만든다. 생성 크기는 `1행 × 1열`이며 사용자가 편집창 안에서 필요한 행과 열을 추가한다.

### 구조 도구 모음

표 편집 대화상자에 다음 버튼을 배치한다.

- `위에 행 추가`
- `아래에 행 추가`
- `왼쪽에 열 추가`
- `오른쪽에 열 추가`
- `선택 행 삭제`
- `선택 열 삭제`
- `셀 병합`
- `셀 분할`
- `폭 자동 맞춤`

행·열 추가는 현재 선택 셀을 기준으로 한다. 여러 셀이 선택되었을 때 행·열 삭제는 선택 영역이 포함한 모든 고유 행 또는 열을 한 번에 삭제한다.

표 전체 삭제는 기존 표 카드의 `표 삭제` 기능을 사용한다. 구조 편집기에서는 마지막 한 행이나 한 열을 제거할 수 없다.

### 병합과 분할

병합은 하나의 연속된 직사각형 선택 영역만 허용한다. 이미 병합된 영역과 일부만 겹치거나, 여러 병합 영역을 가로지르는 선택은 거부한다.

병합 시 각 원래 셀의 텍스트를 병합 시작 셀의 복원 metadata에 좌표와 함께 보존한다. 화면에 표시하는 병합 셀 텍스트는 비어 있지 않은 원래 셀 텍스트를 행 우선 순서로 줄바꿈해 결합한다.

분할은 병합 시작 셀을 선택했을 때만 가능하다. 저장된 복원 metadata가 있으면 각 셀의 원래 텍스트를 복구하고, 없으면 병합 셀 텍스트를 시작 셀에 남긴 채 나머지 셀을 빈 값으로 복구한다.

병합 영역과 교차하는 행·열 추가·삭제는 구조 손상을 막기 위해 차단하고 먼저 해당 셀을 분할하도록 안내한다.

### 열 너비

가로 헤더는 interactive resize 모드로 설정한다. 사용자가 열 경계를 드래그하면 현재 모든 열의 픽셀 폭을 합계 `1.0`인 비율로 정규화하여 저장한다.

`폭 자동 맞춤`은 편집기 표시 폭을 셀 내용에 맞추고 `layout.width_mode`를 `auto`로 되돌린다. 이후 DOCX 출력에서는 저장된 수동 비율 대신 내용 기반 계산을 사용한다.

## 데이터 모델

DB schema는 변경하지 않고 기존 `question_format_json` 및 `choice_format_json` schema-v2를 확장한다.

```json
{
  "schema_version": 2,
  "tables": [
    {
      "id": "table-1",
      "rows": [
        ["구분", "설명"],
        ["A", "긴 설명"]
      ],
      "cells": [
        {
          "row": 0,
          "col": 0,
          "text": "구분",
          "row_span": 1,
          "col_span": 2,
          "horizontal_alignment": "center",
          "vertical_alignment": "center",
          "merge_backup": [
            {"row": 0, "col": 0, "text": "구분"},
            {"row": 0, "col": 1, "text": "설명"}
          ]
        }
      ],
      "column_widths": [0.28, 0.72],
      "layout": {
        "width_mode": "manual"
      }
    }
  ]
}
```

### 폭 모드

- `manual`: 사용자가 편집기에서 드래그한 폭. DOCX 자동 계산보다 우선한다.
- `source`: PDF 파싱에서 복원한 원본 열 비율. 수동 폭이 없을 때 우선한다.
- `auto`: 셀 내용을 기반으로 DOCX 출력 시 계산한다.

이전 payload에 `layout.width_mode`가 없으면 `column_widths`가 유효할 때 `source`, 없을 때 `auto`로 해석한다. 모든 폭은 양수이고 합계가 `1.0`이 되도록 정규화한다. 유효하지 않은 폭은 제거하고 `auto`로 복구한다.

## 구조 변경 규칙

`rows`는 항상 동일한 열 수를 가진 직사각형 배열로 정규화한다. `cells`에는 병합 시작 셀과 개별 셀의 좌표·span·정렬을 저장한다.

- 행 추가: 새 빈 행을 삽입하고 아래쪽 셀 좌표를 1 증가시킨다.
- 열 추가: 모든 행에 빈 셀을 삽입하고 오른쪽 셀 좌표를 1 증가시킨다.
- 행 삭제: 대상 행을 제거하고 아래쪽 좌표를 감소시킨다.
- 열 삭제: 대상 열을 제거하고 오른쪽 좌표를 감소시킨다.
- 열 추가 시 새 열 폭을 `1 / (기존 열 수 + 1)`로 두고 기존 열 비율 전체를 `기존 열 수 / (기존 열 수 + 1)`만큼 축소한다.
- 열 삭제 시 삭제 폭을 제거하고 남은 폭을 다시 정규화한다.

병합 영역과 교차하는 구조 변경은 수행하지 않는다. 이 규칙으로 span을 암묵적으로 확대·축소하면서 생길 수 있는 데이터 손상을 피한다.

## DOCX 자동 폭 계산

### 가용 폭

- 기본 2단 표 폭: 약 82mm
- 자동 1단 표 폭: 페이지 폭 210mm에서 좌우 여백 15mm씩을 뺀 약 180mm

### 내용 폭 점수

각 셀의 줄별 표시 폭을 계산해 가장 긴 줄을 해당 셀의 필요 폭으로 사용한다.

- 한글·CJK·전각 문자: 2단위
- 영문·숫자·일반 기호: 1단위
- 공백과 좁은 구두점: 0.5단위
- 명시적 줄바꿈: 줄별로 따로 계산

병합 셀의 점수는 포함된 열들에 현재 비율 또는 균등 비율로 나누어 반영한다. 각 열은 모든 행에서 받은 점수의 최댓값을 사용한다.

9pt 글꼴 기준 필요 폭은 표시 폭 1단위당 1.6mm로 환산한다. 예상 줄 수는 각 명시적 줄의 필요 폭을 배정된 열 폭으로 나눈 값의 올림으로 계산한다.

### 폭 배분

1. `manual`이고 유효한 `column_widths`가 있으면 그대로 적용한다.
2. `source`이고 유효한 폭이 있으면 원본 비율을 적용한다.
3. 그 외에는 내용 점수에 따라 비율을 계산한다.
4. 각 열은 최소 12mm를 확보한다. 열이 둘 이상이면 한 열의 최대 비율을 전체 가용 폭의 75%로 제한한다.
5. 제한 후 합계를 `1.0`으로 다시 정규화한다.

Word 표에는 계산된 실제 폭을 적용하고 자동 맞춤을 끈다. 표 글꼴은 현재 9pt를 유지하며 폭을 맞추기 위해 더 작게 축소하지 않는다.

### 1단 자동 전환

다음 중 하나이면 해당 표 앞뒤에 연속 구역 나누기를 넣어 1단 폭으로 출력한다.

- 2단 폭에서 모든 열에 최소 12mm를 충족하지 못함
- 내용 점수 기준 예상 줄바꿈이 한 셀에서 3줄을 초과함
- 저장된 `layout.wide`가 참

단순히 열 개수가 5개 이상이라는 현재 규칙은 보조 조건으로만 유지한다. 1단에서도 계산된 폭을 고정 적용하고 표 뒤에서 다시 2단으로 복원한다.

## 오류 처리

- 마지막 행 또는 열 삭제: 작업을 거부하고 최소 크기 안내
- 비직사각형 병합 선택: 작업 거부
- 병합 영역과 일부만 겹치는 선택: 작업 거부 및 분할 안내
- 병합 시작 셀이 아닌 위치에서 분할: 작업 거부
- 손상된 rows/cells/span: 저장 전에 범위 내 직사각형 구조로 정규화하고 복구할 수 없는 병합 metadata만 제거
- 유효하지 않은 폭: 균등 폭 또는 내용 기반 자동 폭으로 복구
- DOCX 폭 계산 실패: 균등 폭으로 fallback하고 exporter 경고에 기록

표 하나의 오류 때문에 전체 모의고사 DOCX 생성을 중단하지 않는다.

## 구현 경계

표 구조 연산과 폭 계산을 GUI 및 DOCX exporter에서 분리한 공통 모듈로 둔다. 공통 모듈은 Qt나 python-docx 객체를 받지 않고 순수 dict/list만 처리한다.

- 표 구조 연산: 삽입, 삭제, 병합, 분할, 유효성 검사, 폭 정규화
- 표 레이아웃 계산: 문자 표시 폭, 열 점수, 2단·1단 판단, 최종 비율
- GUI: 선택 영역과 헤더 픽셀 폭을 공통 모듈의 입력으로 변환
- DOCX exporter: 계산된 비율과 폭을 OpenXML 표에 적용

이 경계로 구조 연산과 출력 계산을 GUI 없이 단위 테스트할 수 있다.

## 테스트 전략

### 구조 단위 테스트

- 네 방향 행·열 추가
- 여러 행·열 동시 삭제
- 마지막 행·열 삭제 차단
- 직사각형 병합과 `merge_backup` 생성
- 분할 후 원문 셀 텍스트 복원
- 기존 병합 영역과 충돌하는 구조 변경 차단
- 열 추가·삭제 후 폭 비율 합계 `1.0`
- 손상된 payload 정규화와 이전 schema-v2 호환

### GUI 테스트

- 표 편집창의 모든 구조 버튼 노출
- 선택 셀 기준 행·열 추가 위치
- 선택 행·열 삭제
- 병합·분할 후 화면과 payload 일치
- 헤더 드래그 폭 저장 및 편집창 재실행 시 복원
- `폭 자동 맞춤` 후 `width_mode = auto`
- 기존 1셀 `<보기>` 표를 다중 셀로 확장

### DOCX 테스트

- 수동 폭이 자동 폭보다 우선
- 원본 폭이 자동 폭보다 우선
- 한글 장문, 영문·숫자, 짧은 제목의 내용 점수
- 병합 셀 점수 분배
- 82mm 2단 유지와 180mm 1단 전환
- OpenXML의 실제 grid column 폭과 `w:tblLayout` 고정값
- 표 뒤에서 2단 구역 복원
- 계산 실패 시 균등 폭 fallback과 경고

### 회귀 테스트

- 기존 `<보기>` 표 75개 payload 유지
- PDF 파싱 다중 셀 표 유지
- 표 없는 문항 및 선택지 저장 회귀 없음
- Mounted 문제은행 write routing 회귀 없음
- 전체 pytest 실행

## 완료 조건

- 사용자가 1셀 표에서 행과 열을 독립적으로 추가하여 임의의 직사각형 표를 만들 수 있다.
- 행·열 삭제, 셀 병합·분할이 데이터 손실 없이 동작한다.
- 열 너비 드래그 결과가 저장·재편집·DOCX 출력에 유지된다.
- 수동 폭이 없는 표는 내용에 맞는 열 비율을 자동 계산한다.
- 좁은 2단에 맞지 않는 표만 자동으로 1단 폭으로 전환한다.
- 기존 schema-v2 표와 `<보기>` 75개 문항에 회귀가 없다.

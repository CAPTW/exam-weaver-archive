# 문제 관리 검색 행 레이아웃 설계

## 목표

문제 관리 탭의 태그·문항·선지 검색 입력창이 필터와 관리 버튼 사이에서 지나치게 좁아지지 않도록 검색 UI를 독립된 두 번째 행으로 이동한다. 사용자가 입력 후 바로 실행할 수 있도록 `조회` 버튼은 검색창의 오른쪽에 인접 배치한다.

## 레이아웃

- 첫 번째 행은 `문제 관리` 제목, EXAM 필터, SUBJECT 필터, `문제 추가`, `서술형 추가`, `오류 검사`, `선택 삭제`로 구성한다.
- 두 번째 행은 검색 입력창과 `조회` 버튼만 포함한다.
- 검색 입력창은 두 번째 행의 남는 가로 폭을 모두 차지하도록 stretch factor 1을 적용한다.
- `조회` 버튼은 검색 입력창 바로 오른쪽에 고정 폭으로 배치하고, 두 위젯 사이에는 기존 레이아웃의 기본 간격만 둔다.
- 검색 입력창의 최소 폭은 320px로 두어 창 폭이 줄어들어도 검색어를 확인할 수 있게 한다.
- 테이블은 두 번째 검색 행 아래에 배치한다.

## 동작

검색 로직과 이벤트 연결은 변경하지 않는다. Enter 키와 `조회` 클릭은 모두 기존 `load_data()`를 호출하며 EXAM·SUBJECT·태그/텍스트 필터 조합도 그대로 유지한다.

## 검증

PyQt 레이아웃 테스트에서 다음을 확인한다.

- 검색창과 `조회` 버튼이 기존 header layout에 포함되지 않는다.
- 두 위젯이 별도 search layout에서 연속된 순서로 배치된다.
- 검색창에 stretch factor 1과 최소 폭 320px 이상이 적용된다.
- vertical layout의 순서가 header, search, table이다.
- 기존 검색·문제 추가·서술형 추가·Mounted repository 관련 테스트가 모두 통과한다.
